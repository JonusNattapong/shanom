<effort_estimation_framework>
Estimate remediation effort using these guidelines:

**Complexity: Low**
- Estimated Time: 2-8 hours
- Skills Required: Junior developer
- Changes: Configuration changes, input validation additions
- Testing: Unit tests sufficient
- Risk of Breaking Changes: Minimal

**Complexity: Medium**
- Estimated Time: 1-3 days
- Skills Required: Mid-level developer
- Changes: Component-level refactoring, library updates
- Testing: Integration testing required
- Risk of Breaking Changes: Moderate

**Complexity: High**
- Estimated Time: 3-5 days+
- Skills Required: Senior developer / Security engineer
- Changes: Architecture changes, authentication redesign
- Testing: Full regression testing required
- Risk of Breaking Changes: Significant

**Complexity: Critical**
- Estimated Time: 1-2 weeks+
- Skills Required: Senior team, possible external consultation
- Changes: Framework-level changes, major refactoring
- Testing: Comprehensive testing, security validation
- Risk of Breaking Changes: High - requires careful rollout
</effort_estimation_framework>

- Best practice compliance

## Prevention Guidelines

### Secure Coding Patterns by Category

#### SQL Injection Prevention
```
❌ VULNERABLE: 
query = "SELECT * FROM users WHERE id = " + userInput

✅ SECURE:
query = "SELECT * FROM users WHERE id = ?"
db.execute(query, [userInput])
```
**Pattern**: Always use parameterized queries/prepared statements

#### XSS Prevention
```
❌ VULNERABLE:
element.innerHTML = userInput

✅ SECURE:
element.textContent = userInput
// OR use auto-escaping template engine
```
**Pattern**: Output encoding or context-aware auto-escaping

#### Authentication Issues
```
❌ VULNERABLE:
if (password === storedPassword) { login() }

✅ SECURE:
if (bcrypt.compare(password, hash, { constant_time: true })) { login() }
// Plus: Rate limiting, MFA, session management
```
**Pattern**: Use established libraries, never roll your own crypto

#### Authorization/IDOR
```
❌ VULNERABLE:
app.get('/api/orders/:id', (req, res) => {
  Order.find(req.params.id).then(order => res.json(order))
})

✅ SECURE:
app.get('/api/orders/:id', authenticate, (req, res) => {
  Order.findOne({ id: req.params.id, userId: req.user.id })
    .then(order => {
      if (!order) return res.status(404).json({ error: "Not found" })
      res.json(order)
    })
})
```
**Pattern**: Always verify ownership/permission

#### SSRF Prevention
```
❌ VULNERABLE:
fetch(userSuppliedUrl)

✅ SECURE:
const ALLOWED_DOMAINS = ['api.internal.com', 'cdn.example.com']
const url = new URL(userSuppliedUrl)
if (!ALLOWED_DOMAINS.includes(url.hostname)) {
  throw new Error("URL not in allowlist")
}
fetch(url.toString())
```
**Pattern**: URL validation, allowlist approach, disable unnecessary URL schemes
